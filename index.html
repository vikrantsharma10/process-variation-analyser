THE DIAGNOSTIC ENGINE

Seven stages. Runs in sequence.
Each hands results to the next.

01 /
PROCESS DECOMPOSITION
Breaks each region's input into discrete comparable steps. Normalises the data before anything else runs.
02 /
VARIANCE ANALYSIS
Compares step count and TAT across regions. Flags the outlier market with the largest deviation from the median.
03 /
ROOT CAUSE ANALYSIS
Assigns a reason code per divergence; Regulatory; Cultural; Resource-Driven; or Legacy Workaround. Runs internally.
04 /
CAGE FRAMEWORK
Tests each reason code against Cultural; Administrative; Geographic; Economic filters. Passes = legitimate local variation. Fails all four = friction.
05 /
STEP CLASSIFICATION
Stamps each step with a verdict; Standardise; Modify; or Keep Local. This is the divergence map the process owner acts on.
06 /
THEORY OF CONSTRAINTS
Identifies the single binding bottleneck. Suppresses non-constraint fix recommendations until the bottleneck is resolved first.
07 /
PARETO FILTER
Surfaces the 20% of fixes delivering 80% of improvement. Splits output into Priority Actions P1–P4 and Secondary Actions.
OUTPUT /
TWO AUDIENCES
Process Owner gets operational language and a fix sequence. Leadership gets commercial framing and a TAT opportunity. Both capped at 4 bullet points.
WHAT YOU GET
